# Сегментация пользователей интернет-магазина

Проект по кластеризации пользователей на основе поведенческих данных (просмотры, корзины, покупки). Цель - выделить группы пользователей со схожим поведением для персонализации маркетинга.

**Ссылка на датасет:** [Kaggle - eCommerce behavior data from multi category store](https://www.kaggle.com/datasets/mkechinov/ecommerce-behavior-data-from-multi-category-store/data)

---

## Этапы работы

**Технологии:** Python, Pandas, Scikit-learn, KMeans, Matplotlib, Seaborn

### 1. **Загрузка и подготовка данных**
   - Загрузка 10 млн строк за ноябрь 2019
   - Типы событий: просмотры (97%), корзины (1.5%), покупки (1.4%)
   - Заполнение пропусков в `category_code` и `brand` значением 'unknown'

### 2. **Feature Engineering (признаки для каждого пользователя)**
   - **Счётчики событий:** `total_views`, `total_purchases`, `total_carts`
   - **Финансовые признаки:** `avg_price_purchased`, `total_spent`
   - **Разнообразие интересов:** `unique_categories`, `unique_brands`
   - **Временные признаки:** `first_event`, `last_event`, `active_days`
   - **Производные признаки:** `conversion_rate`
   - Фильтрация неактивных пользователей: оставлены только с ≥5 событиями (1.1M → 500k пользователей)

### 3. **Кластеризация (KMeans)**
   - Масштабирование признаков (StandardScaler)
   - Выбор числа кластеров: метод локтя + силуэт → k=3
   - Итоговая кластеризация на 500k пользователей

## Визуализация кластеров (PCA)

График показывает проекцию 9-мерных данных на 2 главные компоненты. Основная масса пользователей (кластер 0) образует плотное облако, кластеры 1 и 2 расположены на периферии, что подтверждает их особое поведение.

Логарифмическая шкала позволяет видеть одновременно маленькие и большие значения.

- **Красный** - высокое значение, **синий** - низкое.
- Кластер 1 (VIP) выделяется по `total_spent` и `avg_price_purchased`
- Кластер 2 (Активные посетители) - по `total_views`

## График силуэта

В данных 81% пользователей - «пассивные зрители» (почти без покупок). Они очень похожи друг на друга и образуют плотное ядро. VIP и активные пользователи - это небольшие группы с отличным от ядра поведением. Для таких несбалансированных данных низкий силуэт - норма.

---

## Что я узнала

1. **Дисбаланс кластеров:** 81% пользователей - «пассивные зрители», что отражает реальное поведение в e-commerce (большинство просто смотрит).
2. **Feature engineering определил успех:** конверсия и активные дни помогли чётко разделить группы.
3. **Silhouette низкий из-за дисбаланса:** для таких данных это нормально, важнее бизнес-интерпретация.
4. **PCA показал структуру:** маленькие кластеры (VIP и активные) видны как выбросы на фоне основной массы.

**Ключевой вывод:** Даже с низким силуэтом кластеры имеют чёткий бизнес-смысл и могут использоваться для персонализации маркетинга.
